CI/CD with AWS Infrastructure Automation - Implemented GitHub Actions pipelines that can run tests, rebuild infrastructure, or update files directly on EC2 (Dockerized app) or S3 (static hosting) without tearing down the whole stack, demonstrating practical DevOps and cloud automation skills.


Hands-on Infrastructure Experience - Experimented with both EC2 and S3 for hosting: I run the site on EC2 to demonstrate infrastructure management and Docker deployment, while also building workflows for S3 as a way to learn scalable static hosting.

  • For the EC2 setup, I hosted my Dockerized portfolio on EC2 to practice full infrastructure management. The environment includes private subnets across two Availability Zones, fronted by an Application Load Balancer in the public subnets for traffic distribution, with Route 53 handling DNS. Terraform provisions the stack, with state stored in S3 and locked by DynamoDB. This design demonstrates end-to-end AWS skills in networking, automation, and deployment beyond static site hosting.
  • For the S3 setup, the website content is stored in S3 with an OAC so it is not publicly exposed, and served through CloudFront, which provides caching, SSL termination, and global distribution. I integrated ACM with CloudFront to enforce HTTPS for secure access, while Route 53 handles DNS with an alias target to CloudFront for a clean custom domain. As with my EC2 setup, Terraform provisions all resources, with state stored in S3 and locked by DynamoDB. I chose this setup as my actual hosting solution because it delivers reliability and scalability at a much lower cost, making it ideal for a static portfolio site.
